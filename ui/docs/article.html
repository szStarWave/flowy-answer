微信小程序 登录态 接入问题 好问题

我们在开发微信小程序时，需要接入兔小巢的反馈功能，但是遇到了登录态传递的问题。按照文档说明，我们尝试了多种方式，但似乎都无法正确传递用户的登录信息。

具体场景是这样的：

  • 用户在我们的小程序中已经登录
  • 跳转到兔小巢反馈页面时，希望保持登录状态
  • 但发现每次进入都是未登录状态

我们尝试过的方法：

  1. 在 URL 中传递 token 参数
  2. 使用本地存储保存登录信息
  3. 调用兔小巢的登录接口

请问官方是否有标准的接入方式？或者小程序本身就不支持传递登录态？

补充信息：我们使用的是兔小巢小程序 SDK v2.1.0，微信基础库版本 2.30.0

传递登录态是针对 app 传递的，小程序默认是微信登录态。

关于小程序登录态的问题，这里统一回复一下。兔小巢小程序接入时，默认使用的是微信登录态，不需要额外传递 app 的登录态。如果需要关联用户身份，可以在小程序端调用 txc.login() 方法，我们会返回微信的 openid，你可以用这个来关联你的用户系统。

具体接入文档可以参考：小程序接入指南

1. 兔小巢是支持用户以游客的匿名身份登录进行反馈的，图中该用户属于这种情况。这种情况下，官方是无法帮您获取用户的昵称头像等信息的，望理解~

2. 可以收集用户信息，让用户填写联系方式，在管理后台-用户信息收集中设置。
